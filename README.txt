26' 초대교회 예배여행 — 세미나 접수 사이트 설치 안내
====================================================================

[포함된 파일]
  index.html         신청 페이지 (메인)
  admin.html         관리자 페이지 (실시간 접수현황)
  css/style.css       디자인 스타일
  js/config.js        설정 파일 (★ 반드시 값 채워넣기)
  js/script.js        신청 폼 로직
  js/admin.js         관리자 페이지 로직
  code.gs             구글 시트에 붙여넣을 백엔드 스크립트
  README.txt          이 안내 파일


====================================================================
STEP 1. 구글 스프레드시트에 Apps Script 연결하기
====================================================================
1. 아래 스프레드시트를 엽니다.
   https://docs.google.com/spreadsheets/d/1amITd9gMte_mbhaf_bjqu6yaXSL_5LXNo2kJq7-StxI/edit

2. 접수 데이터가 쌓일 시트 탭의 이름을 "접수현황" 으로 만들어 주세요.
   (이미 만들어진 gid=1496045290 탭을 사용하셔도 되고, 새 탭을 추가해도 됩니다.
    단, 이름은 반드시 "접수현황" 이어야 code.gs 가 정상 동작합니다.
    다른 이름을 쓰고 싶다면 code.gs 상단의 SHEET_NAME 값을 원하는 이름으로 바꿔주세요.)

3. 메뉴에서 [확장 프로그램] > [Apps Script] 를 클릭합니다.

4. 기본으로 열려 있는 코드를 모두 지우고, 첨부된 code.gs 파일의 내용을
   전체 복사해서 붙여넣습니다.

5. 저장(디스크 아이콘)합니다.


====================================================================
STEP 2. 웹앱으로 배포하기
====================================================================
1. Apps Script 화면 우측 상단의 [배포] > [새 배포] 클릭

2. 유형 선택(톱니바퀴 아이콘)에서 "웹 앱" 선택

3. 아래와 같이 설정합니다.
     - 설명 : 초대교회 예배여행 접수 연동 (원하는 이름)
     - 다음 사용자 권한으로 실행 : 나(본인 계정)
     - 액세스 권한이 있는 사용자 : 전체(익명 사용자 포함)
     ※ "액세스 권한이 있는 사용자"를 반드시 "전체"로 설정해야
        신청 페이지/관리자 페이지에서 접근할 수 있습니다.

4. [배포] 클릭 → 구글 계정 권한 승인 절차를 진행합니다.
   (Google에서 "확인되지 않은 앱"이라는 경고가 나오면
    "고급" > "이동(안전하지 않음)" 을 눌러 진행하시면 됩니다.
    본인이 직접 만든 스크립트이므로 안전합니다.)

5. 배포가 완료되면 "웹 앱 URL" 이 표시됩니다.
   예시: https://script.google.com/macros/s/AKfycb.../exec
   이 주소를 복사해두세요.

   ※ 이후 code.gs 를 수정하게 되면, [배포] > [배포 관리] 에서
      기존 배포를 "편집"하고 새 버전으로 저장해야 반영됩니다.

STEP 3. js/config.js 설정하기
====================================================================
js/config.js 파일을 열어 아래 값을 채워주세요.

  1) GAS_WEB_APP_URL
     → STEP 2 에서 복사한 웹 앱 URL 을 붙여넣습니다.

  2) AES_SECRET_KEY
     → 개인정보 암호화에 쓰이는 비밀키입니다. 원하는 문자열로 변경하세요.
       (예: "dreamnanum-2026-실제운영키-abcdef")
       ⚠ 이 값을 바꾸면 이전에 저장된 데이터는 새 키로 복호화되지 않습니다.
          운영 시작 전에 한 번만 정하고 이후에는 바꾸지 마세요.

  3) NAVER_STORE_URL
     → 현재 값은 판매자가 상품을 "수정"하는 관리자 링크입니다.
       고객이 실제로 결제할 수 있는 스마트스토어 "상품 상세페이지" 주소로
       교체해주세요. (스마트스토어센터 > 상품관리 > 해당 상품 > "미리보기"
       버튼을 누르면 고객에게 보이는 실제 URL을 확인할 수 있습니다.)

  4) ADMIN_PASSWORD_HASH
     → 관리자 페이지 접속 비밀번호입니다. 기본값은 "dream2026" 입니다.
       변경하려면 브라우저 개발자 도구 콘솔에서 아래처럼 입력해
       해시값을 만든 뒤 붙여넣으세요.

         CryptoJS.SHA256("원하는비밀번호").toString()

       (admin.html 을 한 번 열어두면 CryptoJS 가 로드되어 있어
        브라우저 콘솔에서 바로 실행할 수 있습니다.)

STEP 4. 사이트 열기 / 호스팅
====================================================================
- 압축을 풀어 index.html 을 더블클릭하면 로컬에서 바로 확인 가능합니다.
- 실제 서비스로 사용하려면 폴더 전체를 웹호스팅(예: 카페24, Netlify,
  Github Pages, 교회 홈페이지 서버 등)에 업로드하세요.
- 폴더 구조(css/, js/ 경로)를 그대로 유지해야 합니다.


====================================================================
동작 흐름 요약
====================================================================
1. 신청자가 index.html 에서 이름/성별/생년월/핸드폰번호 입력
2. 입력값은 브라우저에서 AES로 암호화된 뒤 Apps Script 로 전송
3. Apps Script 가 암호화된 값을 구글 시트에 한 행씩 저장
4. 저장이 끝나면 네이버 스마트스토어 결제 페이지가 새 창으로 열림
5. 관리자는 admin.html 에 비밀번호로 접속
6. 관리자 페이지가 Apps Script 로 실시간 조회(GET) → 값을 복호화해 표로 표시
   (30초마다 자동 새로고침, "새로고침" 버튼으로 즉시 갱신도 가능)


====================================================================
참고 / 한계
====================================================================
- 관리자 비밀번호 확인은 브라우저(클라이언트) 단에서 이루어지는
  간단한 방식입니다. 매우 민감한 정보라면 서버 단 인증이 필요하지만,
  요청하신 "간단한 클라이언트 암호화" 수준에 맞춘 구성입니다.
- AES 비밀키는 config.js 안에 코드로 존재하므로, 소스를 볼 수 있는
  사람에게는 노출됩니다. "구글 시트 원본 데이터를 평문으로 남기지
  않기 위한" 용도로는 충분하지만, 완전한 서버측 보안은 아님을
  참고해주세요.
